# Правила верстки сайтов

## Типографика

### Висячие предлоги
Все короткие предлоги, союзы и частицы НЕ должны оставаться в конце строки.
После них ставить неразрывный пробел `&nbsp;`.

**Список слов:** в, во, на, с, со, к, ко, у, о, об, обо, и, а, но, из, по, за, до, от, не, ни, при, для, без, над, под, про, через, или, да

**Пример:**
- ❌ `в условиях` → строка может разорваться после "в"
- ✅ `в&nbsp;условиях` → "в" всегда прилипает к следующему слову

### Тире
- Между словами — длинное тире `—` (не дефис `-`)
- Перед тире и после — неразрывный пробел: `&nbsp;—&nbsp;`

### Кавычки
- Русские кавычки: «ёлочки» — `&laquo;текст&raquo;`
- Не использовать прямые кавычки `"текст"`

## КРИТИЧНО: Как избежать сломанной вёрстки

### Причина прошлых багов
`raw.githubusercontent.com` кеширует файлы на CDN на несколько минут после пуша.
Если скачать файл сразу после пуша — можно получить СТАРУЮ версию, отредактировать её
и запушить обратно, **затерев предыдущий фикс**.

### Правило
1. **Всегда скачивать файл только через GitHub API**, не через raw.githubusercontent.com:
   ```python
   GET https://api.github.com/repos/{repo}/contents/index.html
   # content в base64, декодировать
   ```
2. **После каждого редактирования CSS — проверять баланс скобок** перед пушем:
   ```python
   css.count('{') == css.count('}')
   ```
3. Если правишь несколько свойств подряд в одном блоке (например добавляешь `animation:`),
   всегда явно проверять, что блок закрыт `}` — особенно когда правка — это просто
   замена одной строки на другую без явного добавления `}`.
